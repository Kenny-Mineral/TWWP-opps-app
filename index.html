Current (dev phase): Read-only via API Key. You can list and link files from a public or shared folder. No write access.

Why not Service Account JSON? Service Account credentials contain a private key. Storing it in a browser app exposes it to anyone with DevTools — Google explicitly warns against this. Never paste a service account JSON here.

For read + write access use OAuth 2.0:
1. In Google Cloud Console → APIs & Services → Credentials → Create OAuth 2.0 Client ID (Web Application type)
2. Add your Netlify URL as an Authorised Redirect URI (e.g. https://endearing-empanada-68c8c7.netlify.app)
3. Note the Client ID — safe to put in the app (it is public, not secret)
4. When you click Connect, the user signs in with their Google account and grants the app permission to their Drive
5. No private key is ever in the browser — Google handles the auth
6. The folder just needs to be accessible to whoever signs in — no extra sharing required

Full automated sync (no user login required) needs the Rails backend holding the service account key server-side. Prioritise once the backend exists.

📜
Architecture Decision Records capture every significant choice made about how the platform is built — the problem, what was considered, what was decided, and why. Claude updates these automatically when decisions are made. Use [REF:ADR-xxx] to cross-reference from other entries or pages.
📜
Block Index is a stable map of all major platform systems and sections. IDs like SYS-INF-001 never change even when sections are reorganised. Use them in ADRs and dev notes to create traceable cross-references.
📜
Growth Log records what changed, when, and why. Claude adds an entry at the end of every session where decisions or features changed. This gives any future developer or Claude session a dated history of how the platform evolved.
📜
Dev Notes shows the current build intent, open questions, and implementation notes for each page. These are also visible in the per-page Dev Notes panel (the toggle button on every page). Notes are hardcoded in the DEVNOTES constant and updated by Claude.
